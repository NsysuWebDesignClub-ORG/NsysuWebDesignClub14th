活動摘要

【C++活動內容】

預計進行以 C++ 程式設計為主題的活動，結合 Kahoot 互動問答與實作題練習。首先透過 Kahoot 測試學員對變數、if-else、迴圈等基本概念的掌握，再以實作題提升同學的程式解題能力，藉由實際編碼加深概念理解。

題目舉例說明

C++ Kahoot

題目：給定下列函式 F()，執行 F() 時哪一行程式碼永遠不會被執行到？

選項：
  • A. a = a + 5;
  • B. a = 5;
  • C. a = a + 2;
  • D. 每一行都會執行到
計算過程：
程式從函式 F() 的第一行開始執行，依序執行各指令。模擬程式運行過程中發現， 因為在執行到 if 條件判斷前後邏輯安排，條件式總是評估為 false， 導致 a = 5; 這一行永遠不會被運行。通過逐步跟踪變數的變化，可以確定該行不被觸及。
答案與解說：
正確答案為 B。原因在於程式設計中， if 條件設定保證了 a = 5; 這一行永遠不會滿足執行條件。這個題目讓學生理解如何透過模擬程式運作流程， 來檢查邏輯漏洞及死碼。
教學目標：
本題旨在幫助學生熟悉程式的執行流程及條件判斷， 並進一步學習如何識別並排除不必要的程式碼（死碼）。藉此提升解讀與分析程式邏輯的能力， 進而培養解決實際編程問題的技能。

C++ 實作

阿姆斯壯數題目說明

阿姆斯壯數（又稱自戀數）是指一個 n 位數，其各位數字的 n 次方和恰好等於該數本身。
例如：153 為三位數，可證明 1³ + 5³ + 3³ = 153。

技術點：
1. 利用迴圈與取餘數（%）運算將數字拆解成各個位數。
2. 使用乘方運算（或迴圈計算連乘）計算每個數字的 n 次方。
3. 累加所有次方後，與原數字進行比較判斷。

與上課內容連結：
本題整合了變數操作、基本算數運算、條件判斷與迴圈控制， 是上課中強調的基礎程式設計技巧的實際應用，有助於學生加深對數字處理與邏輯流程的理解。

Bert 三角形題目說明

Bert 三角形題目要求根據使用者輸入的行數，輸出一個特定格式的三角形圖案。 題目設計通常需要依據行數及每列的空格與圖案符號（如星號或數字）的排列， 來呈現對稱或遞增的效果。

技術點：
1. 利用巢狀迴圈，外層迴圈控制行數，內層迴圈負責輸出每行的空格與圖案符號。
2. 條件判斷決定在特定位置輸出空白或圖案元素，進而調整格式與排版。

與上課內容連結：
此題目有效練習了巢狀迴圈與字串格式輸出技巧， 是「迴圈與條件判斷」單元的重要延伸，能夠幫助學生學會設計演算法與控制輸出格式。

web Kahoot

題目：Flask 如果要顯示網頁頁面的話，哪個方法為被使用？

選項：
  • A. redirect
  • B. template
  • C. static
  • D. jinja
答案與解說：
正確答案為 B。Flask 運作時會使用 template 方法配合 Jinja2 模板引擎來渲染和顯示網頁頁面， 而 redirect 則用於重新導向，static 用於提供靜態資源，jinja 雖為模板引擎名稱但並非直接呼叫的方法。 此題目旨在幫助學生了解 Flask 框架核心的網頁渲染機制。
教學目標：
本題目的設計是為了讓學生熟悉 Flask 中模板渲染的基本流程， 認識如何運用 template 方法和 Jinja2 結合，以產生動態網頁。 同時，學生也能學會區分不同方法的用途，鞏固網頁開發的基礎知識。
